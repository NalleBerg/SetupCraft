── ARCHITECTURE: SCROLLBAR REFINEMENTS ─────────────────────────────────

  WM_VSCROLL child-scroll approach (currently in dep_edit_dialog.cpp):
    Moving ALL children works but can cause combo-box dropdowns to be
    positioned incorrectly when the user has scrolled.  The standard Win32
    fix is to use a scrollable container HWND (a plain child STATIC with
    WS_CLIPCHILDREN) and put all controls inside it; scrolling then moves
    just the container.  Mark this as a known limitation for now.

── FEATURE IN PROGRESS — DEP PERSISTENCE (STILL BROKEN) ────────────────

 SYMPTOM:
   User picks a folder in the dep picker, clicks OK, sees it in the
   dep list — but closing and reopening the folder-edit dialog shows
   an empty list.

 ROOT CAUSE (confirmed):
   CollectAllFiles (IDC_COMP_ENABLE auto-populate) creates only
   source_type="file" rows. No source_type="folder" rows are ever
   auto-created.

   addVFS in the dep picker searches for source_type="folder" comps.
   None found → lp = -2 for all folder nodes.

   Picker OK handler only collects items where tv.lParam > 0.
   Folder nodes with lp == -2 are silently dropped from outDeps.
   outDeps is empty → nothing written → deps gone on reopen.

 THE FIX (four changes to mainwindow.cpp):

 CHANGE 1 — CompFolderDepPickerData struct (~line 4421):
   Add a folderSnapMap field:
     std::map<HTREEITEM, const TreeNodeSnapshot*> folderSnapMap;

 CHANGE 2 — addVFS lambda in WM_CREATE (~line 4739):
   After inserting the folder HTREEITEM hNode, add:
     if (hNode) pData->folderSnapMap[hNode] = &snap;

 CHANGE 3 — OK handler in IDC_FDDP_OK (~line 4856):
   After the existing `if (lp > 0) { ... }` block, add:
     } else if (lp == -2) {
       auto it = pData->folderSnapMap.find(hItem);
       if (it != pData->folderSnapMap.end() && it->second) {
         const TreeNodeSnapshot* fs = it->second;
         if (!fs->fullPath.empty() && pData->projectId > 0) {
           int foundId = 0;
           for (const auto& c : s_components)
             if (c.source_type == L"folder" &&
                 _wcsicmp(c.source_path.c_str(), fs->fullPath.c_str()) == 0)
               { foundId = c.id; break; }
           if (foundId == 0) {
             ComponentRow nc;
             nc.project_id  = pData->projectId;
             nc.display_name = fs->text.empty() ? fs->fullPath : fs->text;
             nc.source_type = L"folder";
             nc.source_path = fs->fullPath;
             int newId = DB::InsertComponent(nc);
             if (newId > 0) { nc.id = newId; s_components.push_back(nc); foundId = newId; }
           }
           if (foundId > 0) pData->outDeps.push_back(foundId);
         }
       }
     }

 CHANGE 4 — After picker closes in IDC_FOLDER_DLG_CHOOSE_DEPS (~line 5654):
   After `pData->outDependencyIds = pickerData.outDeps;`, rebuild
   otherComponents so that any newly created folder-type comps appear
   in the dep listbox:
     {
       const std::wstring& exPath =
           pData->excludeNode ? pData->excludeNode->fullPath : L"";
       pData->otherComponents.clear();
       for (const auto& oc : s_components) {
         if (!exPath.empty()) {
           if (oc.source_path == exPath) continue;
           if (oc.source_path.size() > exPath.size() &&
               _wcsnicmp(oc.source_path.c_str(), exPath.c_str(),
                         exPath.size()) == 0 &&
               (oc.source_path[exPath.size()] == L'\\' ||
                oc.source_path[exPath.size()] == L'/')) continue;
         }
         pData->otherComponents.push_back(oc);
       }
     }

 IMPORTANT: The anyUnsaved guard (before picker opens) already ensures
   projectId > 0 when there are unsaved comps, so DB::InsertComponent
   in Change 3 is safe for the normal case.
