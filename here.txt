 [OK]  Scrollbar thumb sync fix — dialogs.cpp (ContentRichEditSubclassProc)
         Root cause: RICHEDIT50W read-only never writes back SCROLLINFO.nPos
         after scrolling, so any post-scroll read returns stale data.
         Fix: pre-scroll delta arithmetic — read GetScrollInfo(SIF_ALL) BEFORE
         CallWindowProcW, compute newPos from the scroll code (SB_LINEUP etc.)
         in line-unit space, let RichEdit scroll natively, then SetScrollPos
         to the pre-computed value. No post-scroll read needed. Works for both
         writable and read-only RichEdits.
         Same fix applied to WM_MOUSEWHEEL handler in PreviewWndProc.

 [OK]  Reusable scrollbar module created: scrollbar.h / scrollbar.cpp
         Pure Win32, no SetupCraft dependencies. API:
           HRESC RESC_Attach(HWND)       — subclasses RichEdit, syncs thumb
           void  RESC_Detach(HRESC)      — restores original proc
           void  RESC_SyncThumb(HRESC)   — manual sync after programmatic scroll
           LRESULT RESC_OnMouseWheel(HRESC, WPARAM) — mousewheel with pre-delta
         CMakeLists.txt updated to include scrollbar.cpp.
         API documented in scrollbar_API.txt; entry added to API_list.txt.
