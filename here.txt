 [OK]  Custom scrollbar Phase 1 — my_scrollbar.h / my_scrollbar.cpp
         Custom Win32 scrollbar window class ("MyScrollbar"), DPI-aware
         layout, double-buffered GDI painting. Rounded track/arrows/thumb,
         triangle arrow glyphs. Proportional thumb from GetScrollInfo.
         MSB_VERTICAL, MSB_HORIZONTAL, MSB_NOHIDE flags.
         Public API: msb_attach / msb_detach / msb_sync.

 [OK]  Dual-attach fix
         Two bars (V+H) can now share a single target window. Separate
         property keys MSB_Target_V / MSB_Target_H prevent second attach
         from capturing Msb_TargetSubclassProc as origProc (infinite
         recursion on scroll). Subclass installed once; both bars
         dispatched from Msb_TargetSubclassProc.

 [OK]  Native bar suppression fix
         ShowScrollBar(FALSE) does not stick on RichEdit — control re-shows
         native bars on scroll and resize. Fixed with EM_SHOWSCROLLBAR
         (0x0460) and re-suppression in WM_SIZE, WM_VSCROLL, WM_HSCROLL.

 [OK]  Bar scrolling with RichEdit content fix
         Bar was child of target → moved with scrolled content. Fixed by
         making bar a child of target's parent (sibling), positioned via
         ClientToScreen + ScreenToClient mapping on every WM_SIZE.

 [OK]  Custom scrollbar Phase 2 — mouse interaction
         Arrow clicks: SB_LINEUP / SB_LINEDOWN / SB_LINELEFT / SB_LINERIGHT.
         Track clicks: SB_PAGEUP / SB_PAGEDOWN. Auto-repeat timer (350 ms
         initial delay, 50 ms repeat). Thumb drag: SetCapture, pixel→pos
         mapping, SB_THUMBTRACK + SB_THUMBPOSITION + SB_ENDSCROLL.

 [OK]  Own mousewheel handler
         WM_MOUSEWHEEL / WM_MOUSEHWHEEL intercepted in the target subclass.
         Sends explicit SB_LINEUP / SB_LINEDOWN commands (honouring
         SPI_GETWHEELSCROLLLINES). Suppresses RichEdit's native wheel
         entirely — RichEdit does not update SCROLLINFO.nPos reliably on
         native wheel. Sub-tick wheelAccum accumulation for smooth
         high-precision devices.

 [OK]  Horizontal bar independent from vertical scroll
         WM_VSCROLL / WM_HSCROLL now update only the relevant axis bar.
         (Previously both bars updated on any scroll, causing horizontal
         thumb to drift when scrolling vertically.)

 [OK]  Standalone test app (test_scrollbar/)
         Three-pane layout: top-left read-only RE + vertical bar,
         top-right writable RE + vertical bar, bottom full-width RE
         + vertical + horizontal bars. Bottom pane uses wide-content
         text (each line ~200 chars) for horizontal scroll testing.
         Built as ScrollbarTest.exe via test_scrollbar/CMakeLists.txt.
