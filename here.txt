KNOWN ISSUE — Dependencies page H-bar (thumb drag not working correctly)
------------------------------------------------------------------------

 [!!] Dependencies page ListView horizontal scrollbar — PARTIALLY WORKING
      Arrow buttons (left/right): work correctly.
      Track-click: works correctly.
      Tilt-wheel: works correctly.
      Thumb drag: content moves right when dragging right, but thumb does
        not visually follow. Cannot drag back left. Blinks at right end.

      Root cause (identified): ListView H-scroll is extremely unreliable.
        ShowScrollBar(FALSE) zeroes SCROLLINFO.nPos. LVM_SCROLL does not
        update it. WM_HSCROLL may not update it when nMax=0. The WM_NCPAINT
        fires synchronously inside LVM_SCROLL before the header repositions.
        Result: no single read point gives a reliable current position.

      Approach taken: MsbCtx.lvHPos tracking + Msb_GetListViewHPos()
        (reads -Header_GetItemRect(col0).left). The header IS the ground
        truth — but WM_NCPAINT fires before header is repositioned, so
        reading it there gives stale value and corrupts delta calculation.
        Guard added to skip sync when our bar has capture, but thumb drag
        is still broken because dragStartPos feeds from lvHPos which drifts.

      What still needs to be done:
        - Thumb drag: fix dragStartPos to always read Msb_GetListViewHPos()
          JUST BEFORE capturing the mouse (drag-start moment only). 
          Then in WM_MOUSEMOVE, compute newPos from dragStartPos + pixel
          delta as now. In Msb_ScrollToPos read header BEFORE LVM_SCROLL
          for delta, then read header AFTER for lvHPos readback.
        - The blink at right end: likely from Msb_ScrollToPos being called
          in a tight loop with delta=0 (already clamped). Add early return
          when delta==0 after clamping.
        - Consider: after WM_LBUTTONUP (drag end), re-read header into
          lvHPos one final time to capture any last-moment clamp.

      Other pages: Files, Components, Registry — H-bars all work perfectly.
      Dependencies is unique in keeping stale lvHPos across scrolls/resizes.
