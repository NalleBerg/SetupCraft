SetupCraft Version 2026.03.04.12

Components page — full implementation (04.03.2026, 12:05):

  • Components page activated: The former stub (case 9) is replaced with a fully functional page. All controls are direct children of the main window (same pattern as the Registry page) so WM_DRAWITEM reaches them without an intermediate container.

  • Add Folder / Add File — modal dialog: A new CompEditDlgProc dialog lets the developer enter a display name, description, required flag, source path (Browse… opens SHBrowseForFolderW for folders or GetOpenFileNameW for files), and destination path.   • Edit component: Selecting a row and clicking Edit (or double-clicking the row) reopens the same dialog pre-filled with existing values. Changes are saved via UpdateComponent.
  • Remove component: Confirmation MessageBox before calling DeleteComponent(id) and reloading the page.
  • ListView — 6 columns: Name, Description, Required, Type (Folder/File), Source Path, Destination Path. Loaded from the database on every page switch.
  • Database — components table: New components table with columns id, project_id, display_name, description, is_required, source_type, source_path, dest_path. Created by InitDb (CREATE TABLE IF NOT EXISTS) so existing databases are upgraded automatically.
  • Database — use_components column: ALTER TABLE projects ADD COLUMN use_components INTEGER DEFAULT 0 migration runs at startup. ProjectRow extended with the new field; ListProjects, GetProject, and UpdateProject all updated.
  • i18n — 27 new keys: All page labels, column headers, dialog field labels, button captions, hint text, confirmation messages, and type strings (comp_*) added to en_GB.txt. Other locale files will show English until translated.
  • WM_DRAWITEM guard extended: The range IDC_COMP_ADD_FOLDER–IDC_COMP_REMOVE added so all four page buttons draw with the custom button renderer.
  • SwitchPage cleanup: s_hCompListView is explicitly destroyed and nulled, and s_components cleared, whenever the page is switched away from.
