User picks files via GetOpenFileName.
For each file, finds the target HTREEITEM (hTargetFolder).
  folderPath = (wchar_t*)tvItem.lParam
  if (!folderPath || wcslen(folderPath) == 0)   Г”ГҘГү VIRTUAL node only
      s_virtualFolderFiles[hTargetFolder].push_back(fileInfo)

Files added onto a virtual folder node are stored in s_virtualFolderFiles and
are captured by SaveTreeSnapshot Г”ГҘГҶ snap.virtualFiles Г”ГҘГҶ SaveTreeToDb Г”ГҘГҶ DB.

REMAINING LIMITATION: Files added onto a real-path folder node (lParam != "")
are NOT stored in s_virtualFolderFiles and will not appear in snap.virtualFiles.
They will be missing from the DB after save.  The workaround is to add files
only to virtual (manually-created) folders, not to folders added via "Add Folder".
(This limitation existed before the fix and is considered a future improvement.)

SaveTreeToDb  (static helper in mainwindow.cpp, before SwitchPage):
  Recursively processes a vector<TreeNodeSnapshot>.
  parentVirtualPath starts as the root label (e.g. "Program Files").
  For each node: myPath = parentVirtualPath + "\\" + snap.text
    InsertFile(id, snap.fullPath, myPath, L"__folder__")
    for each f in snap.virtualFiles:
        InsertFile(id, f.sourcePath, myPath + f.destination, f.install_scope)
    recurse into snap.children with myPath as new parent.

On every visit to the Files tab the page HWND is destroyed and recreated.
The four root nodes (Program Files, ProgramData, AppData (Roaming), AskAtInstall)
are always created fresh.  Then the following priority order applies:

  Priority 1 Г”ГҮГ¶ Session snapshot (hasSnapshot == true)
    Any of s_treeSnapshot_* is non-empty.
    Г”ГҘГҶ s_virtualFolderFiles.clear()
    Г”ГҘГҶ RestoreTreeSnapshot for each root
      (re-creates HTREEITEM nodes + repopulates s_virtualFolderFiles)
    Used during: tab switches within the same session.

  Priority 2 Г”ГҮГ¶ DB rebuild (hasSnapshot == false, project.id > 0)
    DB::GetFilesForProject(id) returns rows.
    Rows are sorted shallowest-first (fewest backslashes in destination_path).
    A map<wstring, HTREEITEM> (pathToNode) tracks virtual path Г”ГҘГҶ live handle.
    Г”ГҮГі install_scope == "__folder__":
        Extract parent path and node name from destination_path.
        AddTreeNode(hTree, parentHandle, nodeName, source_path)
        Register new HTREEITEM in pathToNode.
    Г”ГҮГі anything else (file row):
        Find parent HTREEITEM via pathToNode.
        Push VirtualFolderFile into s_virtualFolderFiles[parentHandle].
    TreeView_Expand called on Program Files root.
    Used during: first open after restart when project has been saved.

  Priority 3 Г”ГҮГ¶ Disk scan fallback (no snapshot, no DB rows)
    if (!s_currentProject.directory.empty())
        PopulateTreeView(hTree, directory, installPath)
    Scans actual disk folders, builds real-path nodes.
    Used during: legacy projects that predate the DB-rebuild fix, or new
    projects that have never been saved.

  Priority 4 Г”ГҮГ¶ Empty tree
    No snapshot, no DB rows, no directory.  Only root stubs shown.

    (Dep Picker population Г”ГҮГ¶ added 2026-03-15)

EnsureTreeSnapshotsFromDb()  (public static MainWindow member, mainwindow.cpp ~5240)

  Called at the start of CompFolderDepPickerDlgProc WM_CREATE to guarantee that
  all four s_treeSnapshot_* vectors are non-empty so the dep picker has data.

  PROBLEM solved: The old implementation had `if (!snapVecs[si]->empty()) continue`
  inside the per-folder loop.  Once the first folder row was added to a section the
  guard fired on every subsequent row, silently dropping all of them.

  NEW ALGORITHM:
    Step 1 Г”ГҮГ¶ Check which sections need rebuilding (only those still empty).
    Step 2 Г”ГҮГ¶ Load DB::GetFilesForProject(id) for the project.
    Step 3 Г”ГҮГ¶ Pass 1: for every "__folder__" row create/update a
             std::map<wstring, TreeNodeSnapshot> nodeMap entry.
             Key = destination_path (full virtual path).
             The map gives O(log n) lookup and stable references.
    Step 4 Г”ГҮГ¶ Pass 2: for every non-folder (file) row derive parentPath
             (strip the last backslash segment from destination_path) and call
             nodeMap[parentPath].virtualFiles.push_back(vf).
    Step 5 Г”ГҮГ¶ Pass 3: sort nodeMap keys deepest-first (most backslashes first).
             For each key: if the node has no parent in nodeMap it is a section
             root Г”ГҮГ¶ append to snapVecs[si].  If it has a parent, move it into
             nodeMap[parentKey].children.  This ensures children are fully
             assembled before being inserted into their parent.

  Result: all subfolders (AskAtInstall, nested WinUpdate children, etc.) and
  all their file entries are correctly populated in the snapshot vectors.

PopulateSnapshotFilesFromDisk()  (free static function, mainwindow.cpp ~5355)

  Signature:
    static void PopulateSnapshotFilesFromDisk(std::vector<TreeNodeSnapshot>& nodes)

  Purpose: For any real-path node (snap.fullPath non-empty) whose virtualFiles
  vector is STILL empty after EnsureTreeSnapshotsFromDb, perform a one-time
  FindFirstFileW scan of direct child files and populate snap.virtualFiles.
  Recurses into children.

  This handles developer machines where real-path disk folders exist but have
  not yet been saved (so no DB file rows exist yet) Г”ГҮГ¶ locale/, img/, WinUpdate
  subfiles etc. all become visible in the dep picker immediately.

  DB-first principle: EnsureTreeSnapshotsFromDb runs first; this function is
  a fallback only.  If the DB already populated virtualFiles it is skipped.

  Called in CompFolderDepPickerDlgProc WM_CREATE, once per section:
    MainWindow::EnsureTreeSnapshotsFromDb();
    PopulateSnapshotFilesFromDisk(s_treeSnapshot_ProgramFiles);
    PopulateSnapshotFilesFromDisk(s_treeSnapshot_ProgramData);
    PopulateSnapshotFilesFromDisk(s_treeSnapshot_AppData);
    PopulateSnapshotFilesFromDisk(s_treeSnapshot_AskAtInstall);
