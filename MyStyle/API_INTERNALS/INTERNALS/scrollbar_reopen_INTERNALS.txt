When the user closes a project (IDM_FILE_CLOSE) the main window is destroyed
with DestroyWindow(hwnd) *without* going through SwitchPage first.  The child
windows (Files TreeView, ListView, Components TreeView/ListView 횚횉짧) receive
WM_DESTROY, which fires Msb_TargetSubclassProc 횚책횈 msb_detach, which frees the
MsbContext heap allocation.  However, the static HMSB handles that SwitchPage
normally clears (s_hMsbFilesTreeV etc.) remain non-NULL and point to the freed
memory.

On the *next* MainWindow::Create call a new window is created and its very
first WM_SIZE fires msb_reposition() on those dangling pointers 횚책횈 heap
corruption 횚책횈 crash (usually within ~1 second of the window appearing).

횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉횚철횉
When you attach custom scrollbars to a new page's TreeView / ListView:

  (a) Declare the four HMSB statics (V+H for each control).
  (b) Add them to the Create() stale-state reset block (plain NULL, no detach).
  (c) Add them to MainWindow::WndProc WM_DESTROY (plain NULL, no detach).
  (d) In SwitchPage 횚횉철 TEARDOWN section 횚횉철 detach them BEFORE the controlIds[]
      loop destroys the target windows.  The detach block must sit directly
      above the "Destroy all known control IDs" comment, NOT after it.
      EVERY page whose TreeView/ListView IDs appear in controlIds[] must have
      its MSBs detached here.  Getting this wrong = double-free crash.
        if (s_hMsbFooV) { msb_detach(s_hMsbFooV); s_hMsbFooV = NULL; }
        횚횉짧
  (e) In SwitchPage 횚횉철 SETUP section (case N) 횚횉철 call msb_attach AFTER the
      target window is fully populated (all InsertItem + Expand calls done).
      This ensures ShowScrollBar(FALSE) inside msb_attach runs after the
      tree has re-enabled native bars for the last time during population.
        s_hMsbFooV = msb_attach(hFoo, MSB_VERTICAL);
        s_hMsbFooH = msb_attach(hFoo, MSB_HORIZONTAL);
        if (s_hMsbFooH)
            msb_set_edge_gap(s_hMsbFooH,
                GetSystemMetrics(SM_CYEDGE) + GetSystemMetrics(SM_CYBORDER) + 6);
        // No extra WM_SIZE or msb_reposition needed after attach 횚횉철 done.
  (f) Add any persistent HWND statics to the Create() stale-state reset block
      and to the explicit exclusion list in the generic child-window destroy loop
      (for IDs outside toolbar range 5010횚횉척5019 / 5080횚횉척5083).
  (g) **REQUIRED**: Create the TreeView / ListView with WS_VSCROLL|WS_HSCROLL
      in its window style.  Without these styles the control never populates
      SCROLLINFO (nMax always 0) 횚책횈 MSB reads "no overflow" 횚책횈 bars never appear.
      The native bar VISUAL is suppressed by msb_attach; the STYLE must remain.
  (h) **REQUIRED for TreeViews**: Hook TVN_ITEMEXPANDED in the page's WM_NOTIFY
      handler.  The GUARD `if (s_hMsbFooV || s_hMsbFooH)` IS MANDATORY 횚횉철 the

  handler on every NC repaint 횚횉철 triggered by expand/collapse, scrollwheel,
