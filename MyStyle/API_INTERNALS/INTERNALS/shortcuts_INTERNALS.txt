The Shortcuts page (SwitchPage index 2) is fully self-contained in:

    shortcuts.h      Ă”Ă‡Ă¶ public API, structs, control IDs, context-menu IDs
    shortcuts.cpp    Ă”Ă‡Ă¶ all implementation + file-scope statics

mainwindow.cpp knows nothing about the page internals.  It only:
  1. Calls SC_Reset()          from Create()
  2. Calls SC_BuildPage()      from SwitchPage case 2
  3. Calls SC_TearDown()       from SwitchPage teardown block
  4. Routes WM_NOTIFY          Ă”Ă¥Ă† SC_OnNotify()
  5. Routes WM_COMMAND         Ă”Ă¥Ă† SC_OnCommand()
  6. Routes WM_CONTEXTMENU     Ă”Ă¥Ă† SC_OnContextMenu()

  Return value: int Ă”Ă‡Ă¶ the absolute Y coordinate (main-window client space) of
  the first pixel below the last control row.  mainwindow.cpp stores this in
  s_scPageContentH and uses it (plus S(15) bottom padding) to compute the
  SCROLLINFO range and decide whether WS_VSCROLL is needed.  Changed from void
  to int to enable the scrollbar feature without adding a global variable.

  SC_BuildPage does NOT create s_hCurrentPage Ă”Ă‡Ă¶ the Shortcuts page controls are

  NOT destroy Shortcuts controls.  SC_TearDown() handles the TreeView and image
  list explicitly; all other controls (statics, buttons, checkbox) are owned by
  hwnd and are destroyed by the generic child-window enumeration in SwitchPage.

  Width is computed to exactly match the three action buttons beneath it:

    const int addW  = S(150), scW = S(140), remW = S(100), btnGap = S(6);
    const int treeW = addW + btnGap + scW + btnGap + remW;  // S(402) at 96 DPI
    const int treeX = (clientWidth - treeW) / 2;            // centred
    const int bRowX = treeX;   // button row left edge == tree left edge

  These constants are defined once near the top of the SM section (before the
  tree CreateWindowExW call) so both treeW and bRowX can reference them.
  Previously the tree was clientWidth*35/200 (~17 %), which was narrower than
  the button row; the two are now pixel-perfect aligned.

  WM_SETFONT is sent to s_hScStartMenuTree with hGuiFont (= s_scaledFont in
  mainwindow.cpp) so the tree text weight and size match the Files/Components
  page trees.

  WM_NOTIFY for TVN_BEGINLABELEDIT and TVN_ENDLABELEDIT requires a specific
  LRESULT return from the window procedure Ă”Ă‡Ă¶ not just 0.  SC_OnNotify therefore
  returns LRESULT and takes a bool* handled out-parameter.

  Caller pattern in mainwindow WM_NOTIFY:
    { bool scH = false; LRESULT scR = SC_OnNotify(hwnd, nmhdr, &scH);
      if (scH) return scR; }

  TVN_BEGINLABELEDIT  return TRUE  Ă”Ă¥Ă† cancels the in-place edit (used for fixed nodes)
                      return FALSE Ă”Ă¥Ă† allows the edit (used for user-added nodes)
  TVN_ENDLABELEDIT    return TRUE  Ă”Ă¥Ă† accepts the new label text
                      return FALSE Ă”Ă¥Ă† rejects the new label (used when pszText is empty)
  TVN_SELCHANGED      return 0     Ă”Ă¥Ă† standard (just enables/disables the Remove button)
