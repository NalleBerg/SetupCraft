H-SCROLL TILT-WHEEL PROGRESS LOG
=================================
Goal: tilt-wheel scrolls ListView content left AND right (Step 2 of H-scroll rewrite)

ROOT CAUSE (confirmed 02.05.2026):
  LVM_SCROLL checks: internal_counter + dx >= 0.
  ShowScrollBar(SB_HORZ, FALSE) zeroes that counter.
  Right-scroll (dx>0): 0 + dx >= 0  → always passes.
  Left-scroll  (dx<0): 0 + dx < 0   → always REJECTED.
  The counter gets zeroed because LVM_SCROLL internally calls
  ShowScrollBar(SB_HORZ, TRUE) which fires WM_SIZE synchronously,
  and our WM_SIZE handler calls Msb_HideNativeBar(H) = ShowScrollBar(FALSE)
  BEFORE the actual scroll happens. Counter is 0 when scroll runs.

ATTEMPTS (all built and tested unless noted):
----------------------------------------------

    Result: right works, left still broken.
    Reason: WM_SIZE runs before WM_NCPAINT inside LVM_SCROLL — guard was in wrong place.

[2] SetScrollInfo(nPos=lvHPos) before LVM_SCROLL(delta) (Apr 29)
    Result: right works, left still broken.
    Reason: SetScrollInfo updates nPos but NOT the internal counter.

    Reason: LVM_SCROLL is a delta-move. Sending newPos moves content by

    Reason: WM_SIZE still fired (and zeroed counter) during the first LVM_SCROLL.
            WM_SETREDRAW does not suppress WM_SIZE.

[9] Guard WM_SIZE Msb_HideNativeBar(H) with !ctxH->inHDeliver (02.05, 09:12)
