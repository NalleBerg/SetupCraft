PROJECT RULES
-------------
- Always compile with makeit.bat, no parameters or piping.
- No magic numbers — all pixel values go through S(), all strings through locale.
- i18n observed at all times: strings in locale/en_GB.txt only until the app is
  ready for publishing; all 19 other locales translated in one pass at that point.
- Everything must look nice — consistent spacing, aligned controls, clean dialogs.
- Simple UI and the simplest way to get things done — no feature creep.
- Comment well — this is open source; the code must be readable by strangers.
- We code so we work in memory and save only when the developer press save!
- We work modular. Every page and dialog gets it's own .h/.cpp./_INTERNALS.txt or _API.txt
  if needed - always check that.
- Every time an _INTERNALS.txt/_API.txt is made API.list.txt must be updated.
- Check API_list for custom ways of doing things often when coding.
- You have GNU coreutils to use when you need it
