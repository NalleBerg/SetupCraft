1. PURPOSE
──────────
  Drop-in custom scrollbar for any SCROLLINFO-based Win32 window, with full
  support for RichEdit controls.  Replace the native bar by calling
  msb_attach(); everything else — sizing, painting, interaction, DPI scaling —
  is handled internally.

  Features
  ─────────
  • Vertical and/or horizontal bar, independently attached.
  • Proportional thumb derived from SCROLLINFO (or measured content for
    RichEdit, because EM_SHOWSCROLLBAR stops the control maintaining nMax/nPos).
  Fade / hidden mode: bar collapses to a 3 px strip when idle, expands on
    cursor proximity.  MSB_NOHIDE keeps the bar fully visible at all times.
  • Click-to-position on the track; auto-repeat on arrow and track hold.
  • Full thumb drag (SB_THUMBTRACK / SB_THUMBPOSITION / SB_ENDSCROLL).
  • Own WM_MOUSEWHEEL handler (respects SPI_GETWHEELSCROLLLINES; sub-tick
    accumulation for precision devices).
  • EM_SETRECT format-rect management for RichEdit: shrinks the usable height
    so the last line is always scrollable fully above the H bar, with a
    breathing-room margin (MSB_VERT_MARGIN, 6 logical px) below it.
  • DPI-aware: all pixel values go through S() internally; works at any DPI.
  • No SetupCraft dependencies — only <windows.h>.

2. INCLUDE AND LINK
───────────────────
  #include "my_scrollbar.h"

  Add my_scrollbar.cpp to the build.  No extra libraries required.


3. PUBLIC API
─────────────

  typedef void* HMSB;

  ── msb_attach ──────────────────────────────────────────────────────────────
  HMSB msb_attach(HWND hTarget, DWORD flags);

    Attach a custom scrollbar to hTarget.

    flags   One or more of:
              MSB_VERTICAL    (0x00)  Bar on the right edge  [default]
              MSB_HORIZONTAL  (0x01)  Bar on the bottom edge
              MSB_NOHIDE      (0x02)  Always fully visible; no fade animation

    Returns an opaque HMSB handle, or NULL on failure.

    Call once per axis.  Attaching both axes to the same target is safe:
      HMSB hV = msb_attach(hWnd, MSB_VERTICAL   | MSB_NOHIDE);
      HMSB hH = msb_attach(hWnd, MSB_HORIZONTAL | MSB_NOHIDE);

    The call automatically:
      • Hides the native scrollbar (EM_SHOWSCROLLBAR / ShowScrollBar) while
        keeping WS_VSCROLL / WS_HSCROLL so SCROLLINFO is maintained.
      • Creates the custom bar as a sibling of hTarget (child of target's
        parent) so it is never scrolled with the target's content.
      • Subclasses hTarget to intercept WM_SIZE, WM_VSCROLL, WM_HSCROLL,
        WM_MOUSEWHEEL, WM_MOUSEHWHEEL, WM_MOUSEMOVE, and WM_DESTROY.
      • For RichEdit targets: applies EM_SETRECT to reserve space below the
        last line (MSB_VERT_MARGIN) and above the H bar.

  ── msb_detach ──────────────────────────────────────────────────────────────
  void msb_detach(HMSB h);

    Detach and destroy the custom scrollbar.
    Restores the native scrollbar and the original window procedure.
    Safe to call with NULL.
    Call from the parent's WM_DESTROY (before child windows are destroyed):

      msb_detach(st->hSbV);  st->hSbV = NULL;
      msb_detach(st->hSbH);  st->hSbH = NULL;

  ── msb_sync ────────────────────────────────────────────────────────────────
  void msb_sync(HMSB h);

    Manually sync the thumb to the current scroll position and repaint.
    Call after any programmatic scroll that bypasses WM_VSCROLL / WM_HSCROLL,
    e.g. after loading new content into a RichEdit that resets the position:

      SendMessageW(hEdit, EM_SETSCROLLPOS, 0, (LPARAM)&pt);
      msb_sync(st->hSbV);

  ── msb_notify_content_changed ──────────────────────────────────────────────
  void msb_notify_content_changed(HMSB h);

    Invalidate the cached content-dimension measurements (richHorzMax,
    richVertMax) and re-sync both the called bar and its peer.
    Call from EN_CHANGE (or equivalent) whenever text is edited so both bars
    reflect the new document dimensions immediately:

      case EN_CHANGE:
          if (st->hSbH) msb_notify_content_changed(st->hSbH);
          break;

    Calling on either bar (V or H) is equivalent — both caches are cleared
    and both bars are synced.

  ── msb_reposition ──────────────────────────────────────────────────────────
  void msb_reposition(HMSB h);

    Re-derive the bar position from the target's current client-rect corners
    and update visibility.  Designed for explicit window-resize events where
    the target's size has changed and the bar must track the new edges.

    Differs from msb_sync in two ways:
      1. Calls Msb_PositionBar (moves the bar window to the new edge location).
      2. Uses the unguarded visibility check (Msb_UpdateVisibility) so the bar
         is hidden if content genuinely fits after the resize — unlike
         msb_sync which also uses Msb_UpdateVisibility — but msb_reposition
         is the correct entry point from WM_SIZE because it also repositions.

    Call from WM_SIZE after EndDeferWindowPos for all four bars:

      msb_reposition(s_hMsbFilesTreeV);
      msb_reposition(s_hMsbFilesTreeH);
      msb_reposition(s_hMsbFilesListV);
      msb_reposition(s_hMsbFilesListH);

    Safe to call with NULL.

  ── msb_set_insets ──────────────────────────────────────────────────────────
  void msb_set_insets(HMSB h, int insetNear, int insetFar);

    Restrict the bar window to a sub-range of the target's edge.
    insetNear : px to skip from the near edge (top for vert, left for horiz).
    insetFar  : px to skip from the far  edge (bottom for vert, right for horiz).
    Use when the target is a top-level window and the bar must not overlap
    fixed chrome (toolbar, status bar):

      msb_set_insets(s_hMsbIdlg, pageY, statusH);

    Triggers an immediate reposition and repaint.

  ── msb_get_bar_hwnd ────────────────────────────────────────────────────────
  HWND msb_get_bar_hwnd(HMSB h);

    Return the bar's HWND.  Use this to exclude the bar from child-moving
    loops — when the target is a top-level window whose scroll handler moves
    all children by -dy, the bar must be skipped or it will drift:

      HWND hMsbBar = msb_get_bar_hwnd(s_hMsbIdlg);
      // ... inside EnumChildWindows or manual loop:
      if (hChild != hMsbBar) { SetWindowPos(hChild, ...); }

  ── msb_set_edge_gap ────────────────────────────────────────────────────────
  void msb_set_edge_gap(HMSB h, int gap);

    Shift the bar inward from the window edge by gap unscaled px.
    Moves a vertical bar left / horizontal bar upward so the hint strip is
    separated from the frame and more visible against the page background:

      msb_set_edge_gap(s_hMsbIdlg, 4);  // 4 px inward on Dialogs page

    Triggers an immediate reposition and repaint.


4. TYPICAL WIRING
─────────────────

  4a. RichEdit with V + H bars (hidden/fade mode)
  ────────────────────────────────────────────────
  // After creating the RichEdit:
  st->hSbV = msb_attach(hEdit, MSB_VERTICAL);
  st->hSbH = msb_attach(hEdit, MSB_HORIZONTAL);

  // In EN_CHANGE (text edited — content dimensions may have changed):
  if (st->hSbH) msb_notify_content_changed(st->hSbH);

  // In WM_DESTROY (before child windows are destroyed):
  msb_detach(st->hSbV);  st->hSbV = NULL;
  msb_detach(st->hSbH);  st->hSbH = NULL;

  4b. RichEdit with V + H bars (always visible — MSB_NOHIDE)
  ────────────────────────────────────────────────────────────
  st->hSbV = msb_attach(hEdit, MSB_VERTICAL   | MSB_NOHIDE);
  st->hSbH = msb_attach(hEdit, MSB_HORIZONTAL | MSB_NOHIDE);

  4c. Top-level page window (vertical bar, restricted to page area)
  ─────────────────────────────────────────────────────────────────
  // On first build of the page:
  if (!s_hMsb)
      s_hMsb = msb_attach(hwnd, MSB_VERTICAL);
  else
      msb_sync(s_hMsb);
  msb_set_insets(s_hMsb, pageY, statusH);  // skip toolbar + status bar
  msb_set_edge_gap(s_hMsb, 4);             // inset 4 px for visibility

  // In SwitchPage / WM_DESTROY — before tearing down children:
  if (s_hMsb) { msb_detach(s_hMsb); s_hMsb = NULL; }

  // In WM_MOUSEWHEEL / WM_VSCROLL child-moving loop — skip the bar:
  HWND hMsbBar = msb_get_bar_hwnd(s_hMsb);
  // EnumChildWindows / manual loop:
  if (hChild != hMsbBar) { SetWindowPos(hChild, ...); }

  // SB_THUMBTRACK — read position from wParam, not GetScrollInfo:
  case SB_THUMBTRACK:
  case SB_THUMBPOSITION: newPos = (int)(short)HIWORD(wParam); break;

  Nothing else is needed.  WM_SIZE, scroll events, and mouse wheel are all
  handled transparently inside the subclass.

  4d. TreeView / ListView (V + H bars, hidden/fade mode)
  ───────────────────────────────────────────────────────
  IMPORTANT — TreeView attach timing:
  msb_attach must be called AFTER all TreeView_InsertItem calls.
  TreeView internally re-enables its native scrollbar during item insertion,
  undoing the ShowScrollBar(FALSE) that msb_attach calls.  Calling attach
  after population means ShowScrollBar(FALSE) runs last and suppresses the
  native bars correctly.

  For the ListView attach there is no such restriction — attach can be called
  after creating the window and adding columns, before populating.  However,
  every PopulateListView / ListView_InsertItem fill path must re-suppress the
  native bars afterward (see re-suppression pattern below).

  // After creating the windows and adding ListView columns:
  s_hMsbListV = msb_attach(s_hListView, MSB_VERTICAL);
  s_hMsbListH = msb_attach(s_hListView, MSB_HORIZONTAL);
  // Same edge gap for the ListView H-bar:
  if (s_hMsbListH) msb_set_edge_gap(s_hMsbListH, GetSystemMetrics(SM_CYEDGE) + 2);

  // After ALL TreeView_InsertItem / expansion calls (tree fully populated):
  s_hMsbTreeV = msb_attach(s_hTreeView, MSB_VERTICAL);
  s_hMsbTreeH = msb_attach(s_hTreeView, MSB_HORIZONTAL);
  // Shift H-bar above the WS_EX_CLIENTEDGE nc border so the hint strip is visible:
  if (s_hMsbTreeH) msb_set_edge_gap(s_hMsbTreeH, GetSystemMetrics(SM_CYEDGE) + 2);

  // After every PopulateListView / batch ListView_InsertItem call:
  // (ListView re-enables its native bars when items are inserted)
  if (s_hMsbListV) { ShowScrollBar(s_hListView, SB_VERT, FALSE); msb_sync(s_hMsbListV); }
  if (s_hMsbListH) { ShowScrollBar(s_hListView, SB_HORZ, FALSE); msb_sync(s_hMsbListH); }

  // In SwitchPage teardown (before DestroyWindow):
  if (s_hMsbTreeV) { msb_detach(s_hMsbTreeV); s_hMsbTreeV = NULL; }
  if (s_hMsbTreeH) { msb_detach(s_hMsbTreeH); s_hMsbTreeH = NULL; }
  if (s_hMsbListV) { msb_detach(s_hMsbListV); s_hMsbListV = NULL; }
  if (s_hMsbListH) { msb_detach(s_hMsbListH); s_hMsbListH = NULL; }

  Implementation notes for ListView and TreeView targets:
  • ListView ignores external WM_VSCROLL/WM_HSCROLL for absolute positioning.
    The module uses LVM_SCROLL (pixel delta) internally via Msb_ScrollToPos.
  • ListView vertical SCROLLINFO.nPos is in rows, not pixels.  Msb_ScrollToPos
    multiplies the row delta by the item row height (ListView_GetItemRect)
    before calling LVM_SCROLL.
  • ListView horizontal SCROLLINFO.nPos is in pixels — no conversion needed.
  • TreeView responds correctly to SetScrollInfo + WM_VSCROLL SB_THUMBTRACK
    during live drag; SB_THUMBPOSITION is sent only on WM_LBUTTONUP.
  • Visibility guard: Msb_UpdateVisibilityGuarded is used for all scroll
    events on TreeView/ListView targets.  It allows FADE_INVISIBLE → FADE_HIDDEN
    (bar becomes visible when content first overflows) but blocks
    FADE_HIDDEN → FADE_INVISIBLE (transient zero scroll-range from internal
    auto-scroll-to-selection cannot hide a bar already showing).
  • H-bar edge gap: call msb_set_edge_gap(hH, GetSystemMetrics(SM_CYEDGE)+2)
    after attaching the H-bar so the 3 px hint strip sits inside the client
    area rather than flush against the WS_EX_CLIENTEDGE nc border.
  • Arrow clicks use LVM_SCROLL (pixel step) for ListView; WM_VSCROLL/HSCROLL
    SB_LINEUP/SB_LINEDOWN for TreeView — handled automatically by Msb_Scroll.
  • msb_notify_content_changed is not needed for TreeView/ListView (their
    SCROLLINFO is always current).

6. RICHEDIT SPECIFICS
──────────────────────
  RichEdit stops maintaining SCROLLINFO (nPos, nMax) when its native bar is
  hidden via EM_SHOWSCROLLBAR.  my_scrollbar works around this:

  Vertical nMax   Measured via Msb_MeasureRichVertMax: EM_GETLINECOUNT +
                  EM_LINEINDEX + EM_POSFROMCHAR + EM_GETSCROLLPOS.  Cached
                  in MsbCtx.richVertMax; cleared on msb_notify_content_changed.

  Horizontal nMax Measured via Msb_MeasureRichHorzMax: scans every line's
                  right edge via EM_POSFROMCHAR(firstChar + lineLen) +
                  EM_GETSCROLLPOS.  Cached in MsbCtx.richHorzMax.

  nPos            Read via EM_GETSCROLLPOS (always accurate even when the
                  native bar is hidden).

  Format rect     EM_SETRECT sets the RichEdit formatting rectangle to
                  (0, 0, w, ch - hBarH - MSB_VERT_MARGIN) so that:
                    • The last line is scrollable fully above the H bar.
                    • MSB_VERT_MARGIN px of white space appear below it.
                  Re-applied after every WM_VSCROLL and WM_HSCROLL because
                  EM_SHOWSCROLLBAR causes RichEdit to reset the format rect.

  Native bar      Suppressed with EM_SHOWSCROLLBAR (not ShowScrollBar —
  suppression     the latter does not stick on RichEdit).  Re-suppressed
                  after every WM_SIZE, WM_VSCROLL, and WM_HSCROLL.
