OVERVIEW
────────
Page index 7.  Lets the user configure a separate test-output location and
trigger a test build of the installer without touching the production output.

Controls are created as direct children of the main window hwnd, at absolute
Y coordinates starting from pageY — identical to the scripts.cpp / settings.cpp
pattern.  No container STATIC is used.

DEFAULT PATH LOGIC
──────────────────
ComputeDefaultTestFolder():
  1. If cfg.outputFolder not empty → cfg.outputFolder + "\test_output"
  2. Else → %USERPROFILE%\Documents\SetupCraft\<ProjectName>-Setupfiles\test_output

ComputeDefaultTestFilename():
  1. If cfg.outputFilename not empty → cfg.outputFilename + "_test"
  2. Else → strip whitespace from GetProjectName() → "<NameNoSpaces>_<ver>_Setup_test"
     e.g. "SetupCraft Suite" v1.2.0 → "SetupCraftSuite_1.2.0_Setup_test"
     Fallback "Setup_test" when name is blank.


UNIQUENESS CHECK
────────────────
UniqueFilename(folder, baseName):
  Checks GetFileAttributesW(folder\baseName.exe).
  If not found → returns baseName unchanged.
  Else iterates baseName(1), baseName(2), … up to (999) until a free slot is found.
  Ensures each test run gets a distinct filename when repeated without cleanup.


SETTINGS PERSISTENCE
────────────────────
Two new fields in SBuildConfig (settings.h):
  testOutputFolder    — DB key "test_output_folder"
  testOutputFilename  — DB key "test_output_filename"

Setters (settings.cpp) call MainWindow::MarkAsModified().
EN_CHANGE on both edits persists immediately via the setters.
TEST_RunTest resolves defaults and persists them before the uniqueness check.


INNO INTEGRATION — NEXT ROUND
──────────────────────────────
TEST_RunTest() currently:
  • resolves folder + filename
  • applies UniqueFilename to avoid overwriting
  • updates IDC_TEST_STATUS to show the resolved output path
  • enables IDC_TEST_OPEN_FOLDER_BTN if the folder already exists
  • does NOT yet compile or launch anything

Next round:
  • Call ISS_GenerateIss() with a test-specific output dir and filename.
  • Launch compile_inno.bat <generated.iss> via ShellExecuteW.
  • On success, optionally launch the resulting .exe.


TEARDOWN
────────
TEST_TearDown() nulls all HWND statics.  Called from SwitchPage before the
controlIds[] DestroyWindow loop.  The controls themselves are destroyed by
the loop (IDC_TEST_* are listed in the controlIds[] array in mainwindow.cpp)
and by the page-area child-window enumeration fallback.
